Version control is important in collaborative projects because it 
allows for group members to work on the same project without 
compromising each other's work by overwriting it. It keeps the 
history of all logs and changes, making it easier to review and 
or restore versions. Version control is important in keeping all
changes safe and free from any conflicts. This helps in the development
of a project organized and collaborative. 

Compared to manual file management, Git makes it easier to track
changes and maintain the history of every version of the project.
It reduces the risk of losing every changes installed on your
project and allows collaborative process between members. Git 
provides tools in comparing changes and restoring previous 
versions when needed.
